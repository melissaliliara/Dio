Como é uma estrutura básica de um HTML?

Voltar

Uma estrutura básica de um documento HTML (Hypertext Markup Language) geralmente segue o seguinte formato:

Aqui está uma breve explicação de cada parte:

<!DOCTYPE HTML> : Essa declaração define o tipo de documento como HTML.
<html>: É o elemento raiz do documento HTML.
<head>: É o cabeçalho do documento, contendo informações sobre o documento, como o título da página, links para arquivos CSS, JavaScript e outros metadados.
<title>: Define o título da página, que é exibido na barra de título do navegador ou na guia da janela.
<body>: É o corpo do documento, onde o conteúdo visível da página é colocado. Isso inclui texto, imagens, links, formulários e outros elementos HTML.
você pode adicionar vários elementos HTML para construir a estrutura e o conteúdo da página, como cabeçalhos (<h1>, <h2>, etc.), parágrafos (<p>), imagens (<img>), links (<a>), listas (<ul>, <ol>, <li>), formulários (<form>, <input>, etc.) e muito mais.
Lembre-se de que esse é apenas um exemplo básico e que um documento HTML completo pode ser muito mais complexo, dependendo das necessidades da página.

Entendendo a diferença entre listas ordenadas e não ordenadas

Voltar

As listas ordenadas e não ordenadas são dois tipos de listas que podem ser criadas em HTML para organizar itens de forma visualmente diferente. Aqui está a diferença entre elas:
1. Listas Não Ordenadas (<ul>):
São representadas pelo elemento <ul>>.
Os itens da lista são precedidos por marcadores, geralmente pontos ou círculos vazados. A ordem dos itens não importa, pois a lista não possui uma sequência específica. Exemplo de código HTML:

Item 2
Item 3

2. Listas Ordenadas (<ol>):
São representadas pelo elemento <ol>.
Os itens da lista são precedidos por números ou letras, indicando uma sequência ordenada.
A ordem dos itens é importante, pois reflete a sequência lógica ou cronológica dos elementos.

Resultado visual:
Item 1
Item 2
Item 3

Ambos os tipos de listas podem conter qualquer conteúdo HTML válido dentro dos itens da lista, como textos, imagens, links e outros elementos. A escolha entre usar listas ordenadas ou não ordenadas depende da intenção e da lógica de organização dos itens a serem apresentados.
